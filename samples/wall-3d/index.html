3D picture wall

A scrolling wall of side-by-side 3D pictures, each woven inline as it enters the viewport and released as it leaves — the lazy-load XRDisplayLayer pattern, and the stress test for the batched one-submit-per-frame weave.

Detecting DisplayXR Browser…
One inline-3d session; an IntersectionObserver creates an XRDisplayLayer per picture on viewport enter and close()s it on leave, so only visible pictures cost weave work. Batched submit (XR_DXR_weave spec v3) weaves all visible rects in ONE runtime call per frame. Get the DisplayXR Browser.
